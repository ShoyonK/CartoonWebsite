About

Team E14’s website allows you to access the most popular anime, characters, and studios with ease. Our website currently has a homepage that lets you easily navigate to the anime, character, staff, and studio listings on our website with our navbar. On each of these pages, we have a carousel listing of cards of the most popular anime, characters, staff members, and studios depending on which page you are on. Clicking on a card takes you to a page with more information about the content of the card so that you can learn more about it.

The purpose of our website is to create an easy to use and visually appealing interface for anyone from new anime fans to anime enthusiasts to learn more about anime.

Our four sources include the AniList API, the Jikan API (the unofficial API for the myanimelist website), the Google Trends API, and the Wikipedia API. The AniList API and Jikan API give us a good number of attributes to choose from for each instance. We were able to mix and match attributes from both these APIs in each of our instance pages for the character and anime instances. The Wikipedia API lets us gather a lot more information on the studios behind the anime, which felt like an important component of information for us so we added it as a fourth model type. The Google Trends API lets us take an anime, character, staff member, or studio name and find other anime-related attributes that are related to the field, which allows users to discover new things about the anime, character, staff member, or studio they are looking at. We believe that by putting together these disparate data sources, we will provide a unique experience to anime fans that isn’t offered elsewhere.

Other information

Data sources:

AniList API: https://anilist.gitbook.io/anilist-apiv2-docs/

Jikan (unofficial MyAnimeList API): https://jikan.moe/

Pytrends (unofficial google trends API): https://pypi.org/project/pytrends/

Wikipedia API (for python): https://pypi.org/project/wikipedia/

Data was obtained through these APIs by dynamically fetching from these APIs depending on what the user clicks on. We took information from these APIs and then linked them together within our own database. We were able to use the documentation of the APIs to easily make calls to them. The Jikan, Pytrends and Wikipedia APIs came with their own python modules which allowed us to access them easily. The AniList API utilized GraphQL to query the API and return a json response type.

Tools:

The following software tools were used in the developement and testing of this website.

Website Development

We used fairly standard tools for the website itself. Flask was the Python framework that was used to build the backend of the website. Bootstrap was the frontend framework used to make our website have an aesthetically pleasing look. JavaScript (and the jQuery library) were used for creating the animated parallax backgrounds, dynamic pagination, carousels, etc. and simple CSS was used for styling.

APIs and Databases

Jikan and Pytrends modules allowed us to access their APIs. We used these modules by creating functions in our backend that called these modules using whatever name, character, staff member, or studio was passed, returning a dictionary of useful information that could be presented on the frontend. The Pytrends API allowed us to collect information regarding related searches for instances on our website. The Wikipedia API allowed us to get more information on the studios in particular. We used MongoDB to store documents queried from the APIs. Our searching, sorting, and filtering was implemented using the functionality provided by MongoDB.

Testing

For testing the backend we used Unittest, the Python testing framework. For testing the GUI we used Selenium. For testing the Javascript and jQuery code, we used the Mocha testing framework.
